求人票サンプル: Webアプリケーションエンジニア

募集ポジション
Webアプリケーションエンジニア

想定企業
BtoB SaaSを提供する成長フェーズのプロダクト企業。営業、カスタマーサクセス、開発チームが連携しながら、業務支援クラウドサービスの機能改善と信頼性向上に取り組んでいます。

業務内容
- Next.js / Reactを用いたフロントエンド機能開発
- Node.js / TypeScriptを用いたAPI設計、実装、テスト
- Cloud Run、Cloud SQL、Cloud LoggingなどGoogle Cloudを活用した運用改善
- GitHub Actionsを用いたCI/CDパイプラインの改善
- パフォーマンス改善、障害調査、再発防止策の実装
- PdM、デザイナー、CSと連携した仕様検討とリリース後の効果検証

必須要件
- Webアプリケーション開発の実務経験2年以上
- React、Vue、Angularなどのモダンフロントエンド開発経験
- Node.js、Go、Python、Javaなどを用いたバックエンド開発経験
- GitHubを用いたチーム開発経験
- ログやメトリクスをもとに不具合や性能課題を調査した経験

歓迎要件
- Google CloudまたはAWSでのアプリケーション運用経験
- CI/CD、テスト自動化、Observability改善の経験
- API設計、DB設計、非同期処理、キャッシュ設計の経験
- 障害対応やポストモーテム作成の経験
- 技術選定や設計方針をチームに説明した経験

期待する役割
単に機能を実装するだけでなく、なぜその設計や技術を選ぶのかを説明し、運用後の品質や開発速度まで考えて改善を進めることを期待します。小さな改善を継続的に届け、チーム全体の開発体験とユーザー体験を高める役割です。

選考で重視する観点
- 技術選定理由を、代替案との比較を含めて説明できるか
- 課題の原因をログ、メトリクス、コード、DBクエリから具体的に特定できるか
- 設計判断において、性能、保守性、運用リスク、開発スピードのバランスを取れるか
- CI/CDやテストを使い、変更の安全性を高められるか
- チームで合意形成しながら改善を進められるか
- 成果をレスポンス時間、エラー率、問い合わせ件数、開発リードタイムなどの指標で語れるか

面接で聞きたいテーマ
- これまで最も難しかった技術課題と、その解決方法
- 技術選定で迷った経験と、最終的な判断理由
- 障害や性能問題をどのように調査し、再発防止につなげたか
- チーム開発で品質や開発速度を改善した経験
- DevOpsやCI/CDを活用して継続的に改善した経験

想定される理想回答の方向性
課題、原因特定、選択肢比較、判断理由、実装、検証、成果、学びが一貫している回答を評価します。特に、ログやメトリクスなどの客観情報に基づき、単発の対応ではなく再発防止や運用改善まで踏み込んでいる回答が高評価です。
